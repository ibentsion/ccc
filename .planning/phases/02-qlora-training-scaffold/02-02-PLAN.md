  <action>
    Create `training/model.py` with `class QLoRALightningModule(pl.LightningModule)`:

    `__init__(self, config: TrainingConfig)`:
    - Store `self.config = config`
    - Resolve compute dtype: `compute_dtype = torch.float16 if config.bnb_4bit_compute_dtype == "float16" else torch.bfloat16`
    - STEP 1 of 3 (TRAIN-03): Load base model with quantization:
      ```python
      bnb_config = BitsAndBytesConfig(
          load_in_4bit=True,
          bnb_4bit_quant_type=config.bnb_4bit_quant_type,
          bnb_4bit_use_double_quant=config.bnb_4bit_use_double_quant,
          bnb_4bit_compute_dtype=compute_dtype,
      )
      base_model = AutoModelForCausalLM.from_pretrained(
          config.model_name,
          quantization_config=bnb_config,
          device_map={"": 0},  # NOT "auto" — avoids DDP conflicts with Lightning
          trust_remote_code=True,
      )
      ```
    - STEP 2 of 3 (TRAIN-03): `base_model = prepare_model_for_kbit_training(base_model)`
    - If gradient_checkpointing: `base_model.gradient_checkpointing_enable(gradient_checkpointing_kwargs={"use_reentrant": False})`
    - STEP 3 of 3 (TRAIN-03): Apply LoRA:
      ```python
      lora_config = LoraConfig(
          r=config.lora_r,
          lora_alpha=config.lora_alpha,
          lora_dropout=config.lora_dropout,
          bias=config.lora_bias,
          target_modules=config.lora_target_modules,
          task_type="CAUSAL_LM",
      )
      self.model = get_peft_model(base_model, lora_config)
      ```

    Do NOT call `self.save_hyperparameters()` — Lightning will crash when the only `__init__` arg is `config` and it's ignored. Hyperparameters are logged in Plan 03 via `task.connect()` instead.

    `forward(self, input_ids, attention_mask, labels=None)`:
    - `return self.model(input_ids=input_ids, attention_mask=attention_mask, labels=labels)`

    `training_step(self, batch, batch_idx)`:
    - `outputs = self(batch["input_ids"], batch["attention_mask"], batch["labels"])`
    - `loss = outputs.loss` (CrossEntropyLoss with ignore_index=-100 is built into HF causal LM — labels with -100 are automatically ignored, implementing TRAIN-06)
    - `self.log("train_loss", loss, on_step=True, on_epoch=True, prog_bar=True, logger=True)`
    - Return loss

    `validation_step(self, batch, batch_idx)`:
    - Same as training_step but log as "val_loss"
    - `self.log("val_loss", loss, on_step=False, on_epoch=True, prog_bar=True, logger=True)`

    `configure_optimizers(self)`:
    - Import `bitsandbytes as bnb`
    - `optimizer = bnb.optim.PagedAdamW32bit(self.parameters(), lr=self.config.learning_rate)` (per research: paged_adamw_32bit)
    - For constant LR (per TRAIN-05): `scheduler = torch.optim.lr_scheduler.ConstantLR(optimizer, factor=1.0, total_iters=0)` — this is effectively no decay
    - Return `{"optimizer": optimizer, "lr_scheduler": {"scheduler": scheduler, "interval": "step"}}`

    `save_adapter(self, path: str)`:
    - `self.model.save_pretrained(path)` — PEFT save_pretrained saves ONLY adapter weights (per TRAIN-07)
    - This produces: adapter_config.json + adapter_model.safetensors (or .bin)

    Imports at top of file:
    ```python
    import torch
    import pytorch_lightning as pl
    from transformers import AutoModelForCausalLM, BitsAndBytesConfig
    from peft import get_peft_model, LoraConfig, prepare_model_for_kbit_training
    from training.config import TrainingConfig
    ```

    CRITICAL: Do NOT use Lightning's BitsandbytesPrecision plugin. Do NOT use device_map="auto".
