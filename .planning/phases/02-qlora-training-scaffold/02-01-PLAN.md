    Create `training/tokenizer.py` with:

    1. `load_tokenizer(model_name: str) -> AutoTokenizer`:
       - Loads `AutoTokenizer.from_pretrained(model_name, trust_remote_code=True)`
       - Sets `tokenizer.pad_token = tokenizer.eos_token`
       - Sets `tokenizer.padding_side = "left"` (decoder-only model requirement)
       - Returns the tokenizer

    2. `class TokenizedCollator`:
       - `__init__(self, tokenizer, max_seq_length: int = 2048)`: stores tokenizer and max_seq_length
       - `__call__(self, batch: list[dict]) -> dict`: receives a list of FIM sample dicts (each with keys: fim_text, prefix, suffix, middle, n_lines)
         - Extracts `fim_text` from each sample
         - Tokenizes with `tokenizer(fim_texts, padding=True, truncation=True, max_length=self.max_seq_length, return_tensors="pt")`
         - Creates `labels` tensor as a COPY of `input_ids`
         - For prompt masking (per TRAIN-06): for each sample in the batch, find the token range corresponding to the "middle" content. The approach:
           a. Tokenize just the middle text: `middle_ids = tokenizer.encode(sample["middle"], add_special_tokens=False)`
           b. Tokenize the prefix part (everything before middle in the FIM structure): `prefix_part = FIM_BEGIN + sample["prefix"] + FIM_HOLE + sample["suffix"] + FIM_END`
           c. `prefix_ids = tokenizer.encode(prefix_part, add_special_tokens=False)`
           d. The middle tokens start at position `len(prefix_ids)` in the non-padded sequence
           e. Account for left-padding offset: `pad_len = (input_ids[i] == tokenizer.pad_token_id).sum().item()`
           f. Set `labels[i, :pad_len + len(prefix_ids)] = -100` (mask padding + prefix/suffix/FIM tokens)
           g. The middle tokens and EOT token keep their real label values
         - Returns dict with keys: `input_ids`, `attention_mask`, `labels` (all torch tensors)

    IMPORTANT: Do NOT import torch at module top level in config.py. Only import torch in tokenizer.py where it's needed.
